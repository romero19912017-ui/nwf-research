# Масштабный план создания экосистемы библиотек NWF

## 1. Цель экосистемы

Создать модульный, высокопроизводительный набор инструментов на основе концепции Нейровесовых Полей (Neural Weight Fields), позволяющий решать широкий спектр задач машинного обучения с акцентом на **инкрементальность**, **встроенную неопределённость** и **калибровку**. Библиотеки должны быть просты в использовании, хорошо документированы и интегрироваться с существующими фреймворками.

---

## 2. Общая архитектура

Вся экосистема строится вокруг **ядра** (core), которое предоставляет базовые абстракции и алгоритмы. На его основе создаются специализированные библиотеки для конкретных областей.

```
nwf-core
├── Charge, Field, Index, Metric, Updater, Calibrator
├── базовые реализации (BruteForce, VAEEncoder)
└── интерфейсы для расширения

Библиотеки-расширения:
├── nwf-vision (CV)
├── nwf-nlp (NLP)
├── nwf-recsys (рекомендации)
├── nwf-ts (временные ряды)
├── nwf-ood (OOD-детекция)
├── nwf-calibration (дополнительные калибраторы)
├── nwf-active (активное обучение)
├── nwf-multimodal (мультимодальность)
└── nwf-experimental (новые методы)
```

Каждая библиотека использует ядро и добавляет специфичные энкодеры, предобученные модели, метрики и примеры.

---

## 3. Перечень библиотек и их функциональность

### 3.1. nwf-core (ядро)
- **Задача**: предоставить фундаментальные классы и алгоритмы NWF.
- **Компоненты**:
  - `Charge` – структура данных `(z, sigma)` с методами сериализации.
  - `Field` – контейнер для хранения зарядов, поддерживающий добавление, удаление, поиск.
  - `Metric` – симметричное расстояние Махаланобиса, евклидово, косинусное.
  - `Index` – интерфейс для индексации; реализации: `BruteForceIndex`, `FAISSIndex` (с поддержкой HNSW, PQ, IVF), `HNSWIndex`.
  - `Updater` – интерфейс для обновления зарядов; реализации: `NoUpdater`, `KalmanUpdater` (EKF), `AmortizedUpdater`.
  - `Calibrator` – калибровка уверенности; `AgreementRatio`, `PlattScaler`, `TemperatureScaling`.
  - `Potential` – вычисление потенциала Φ(r) и его градиента.
  - Вспомогательные утилиты: работа с данными, визуализация, метрики (Precision@k, ECE, AUC).
- **Технологии**: Python, NumPy, SciPy, FAISS (опционально), PyTorch (для некоторых реализаций).
- **Зависимости**: минимальные (numpy, scipy).

### 3.2. nwf-vision (компьютерное зрение)
- **Задача**: инкрементальное обучение для классификации изображений, детекции объектов, сегментации.
- **Компоненты**:
  - Энкодеры на основе свёрточных VAE (или предобученных ResNet, EfficientNet с головой неопределённости).
  - Готовые модели для популярных датасетов (MNIST, CIFAR, ImageNet-subset).
  - Примеры: continual learning на Split-CIFAR, OOD-детекция на SVHN vs CIFAR-10.
  - Интеграция с `nwf-core` для индексации зарядов изображений.
- **Дополнительно**: поддержка аугментаций, пайплайнов обучения.

### 3.3. nwf-nlp (обработка текста)
- **Задача**: инкрементальное обучение для классификации текстов, поиска, тематического моделирования.
- **Компоненты**:
  - Энкодеры на основе трансформеров (BERT, RoBERTa) с дообучаемой головой для `(z, sigma)`.
  - Обёртки для работы с токенизаторами и датасетами (20 Newsgroups, IMDB, sentiment).
  - Поддержка multi-task обучения (одновременная классификация и регрессия).
  - Примеры: добавление новых тем в классификатор новостей без переобучения.

### 3.4. nwf-recsys (рекомендательные системы)
- **Задача**: построение рекомендательных систем, способных инкрементально добавлять новых пользователей и товары.
- **Компоненты**:
  - Энкодеры пользователей и товаров на основе коллаборативной фильтрации (матричная факторизация, нейросети).
  - Гибридные модели, использующие контент (текст/изображения) через `nwf-nlp` или `nwf-vision`.
  - Метрики качества (HR@k, NDCG) с учётом неопределённости.
  - Примеры: MovieLens, Amazon reviews.

### 3.5. nwf-ts (временные ряды)
- **Задача**: прогнозирование, классификация и детекция аномалий на временных рядах с дрейфом концепций.
- **Компоненты**:
  - Энкодеры на основе 1D VAE, LSTM, Transformer.
  - Механизмы онлайн-адаптации через фильтр Калмана.
  - Примеры: ECG, FordA, датчики IoT.

### 3.6. nwf-ood (Out-of-Distribution детекция)
- **Задача**: выявление примеров, не принадлежащих обучающему распределению.
- **Компоненты**:
  - Потенциал Φ(r) как скоринг-функция.
  - Сравнение с современными методами (ODIN, Mahalanobis, Energy-based).
  - Метрики: AUROC, FPR@95.
  - Примеры: MNIST vs Fashion-MNIST, CIFAR-10 vs SVHN.

### 3.7. nwf-calibration (калибровка уверенности)
- **Задача**: предоставить набор калибраторов для любых моделей, но с фокусом на NWF.
- **Компоненты**:
  - Платовское масштабирование, температурный скейлинг, векторный скейлинг.
  - Байесовская калибровка.
  - Оценка калибровки (ECE, MCE, reliability diagrams).

### 3.8. nwf-active (активное обучение)
- **Задача**: выбор наиболее информативных примеров для разметки на основе неопределённости зарядов.
- **Компоненты**:
  - Стратегии: неопределённость (след Σ), максимальное расстояние до ближайшего заряда, потенциал.
  - Интеграция с `nwf-core` для оценки неопределённости.
  - Примеры: активное обучение для классификации изображений.

### 3.9. nwf-multimodal (мультимодальные данные)
- **Задача**: работа с парами данных (изображение+текст, видео+аудио) в едином семантическом поле.
- **Компоненты**:
  - Совместные энкодеры (CLIP-style) с выдачей `(z, sigma)` для каждой модальности.
  - Кросс-модальный поиск с учётом неопределённости.
  - Примеры: поиск изображений по текстовому запросу, дополнение новых пар без переобучения.

### 3.10. nwf-experimental (экспериментальные методы)
- **Задача**: полигон для новых идей (дифференцируемый индекс, связь с гауссовскими процессами, физические аналогии).
- **Компоненты**:
  - Реализации из теоретических работ.
  - Нестабильные, но перспективные алгоритмы.
  - Тесты и бенчмарки.

## 4. План разработки (этапы и milestones)

### Этап 0: Подготовка (1–2 месяца)
- Создать репозиторий `nwf-core`, определить структуру модулей.
- Написать базовые классы `Charge`, `Field`, простейший `BruteForceIndex`.
- Реализовать симметричную метрику Махаланобиса.
- Написать тесты для ключевых функций.
- Подготовить документацию (Sphinx/ReadTheDocs) и пример использования на MNIST.

### Этап 1: Ядро и базовая индексация (2–3 месяца)
- Интеграция с FAISS: `FAISSIndex` с поддержкой L2, косинусной метрики, white-преобразования для глобальной ковариации.
- Реализация кастомной метрики Махаланобиса через предвычисление весов (если возможно) или через модифицированный поиск.
- Добавить `HNSWIndex` и `PQIndex`.
- Реализовать простой `VAEEncoder` на PyTorch.
- Написать `NWFClassifier` на основе индекса.
- Добавить калибраторы: `AgreementRatio`, `PlattScaler`.
- Покрыть код тестами (pytest, coverage >80%).
- Релиз `nwf-core` версии 0.1 на PyPI.

### Этап 2: Специализированные библиотеки (3–6 месяцев)
- **nwf-vision**: обучить VAE на CIFAR-10, реализовать пример continual learning на Split-CIFAR, добавить предобученные веса.
- **nwf-nlp**: обёртка над трансформерами (HuggingFace), пример с 20 Newsgroups.
- **nwf-recsys**: пример на MovieLens, сравнение с ALS.
- **nwf-ts**: пример с ECG, добавить Kalman-обновление.
- Каждая библиотека получает свою документацию, примеры, тесты.
- Релизы версий 0.1 для каждой.

### Этап 3: Продвинутые возможности (6–9 месяцев)
- **Амортизированный энкодер** для быстрого обновления (реализация в `nwf-core` и пример в `nwf-experimental`).
- **Потенциал и OOD-детекция** в `nwf-ood` с бенчмарками.
- **Активное обучение** в `nwf-active`.
- **Мультимодальный поиск** в `nwf-multimodal`.
- Интеграция с ONNX для экспорта энкодеров.
- Профилирование и оптимизация скорости (Cython, GPU).

### Этап 4: Стабилизация и экосистема (9–12 месяцев)
- Полное покрытие документацией всех модулей.
- Создание сайта-лендинга (например, nwf.ai) с описанием, примерами, ссылками.
- Написание серии туториалов (видео, Jupyter notebooks).
- Привлечение сообщества (issue, pull requests).
- Участие в конференциях с докладами о библиотеке.

## 6. Метрики успеха и критерии готовности

- **Качество кода**: покрытие тестами >80%, отсутствие критических багов.
- **Производительность**: скорость поиска и обновления сопоставима с FAISS (для индекса) и с PyTorch (для энкодеров).
- **Документация**: полное описание API, примеры для каждой библиотеки.
- **Сообщество**: >100 звёзд на GitHub, несколько внешних контрибьюторов, использование в сторонних проектах.
- **Научная значимость**: ссылки на библиотеку в статьях, воспроизводимость экспериментов.

## 8. Риски и их mitigation

| Риск | Решение |
|------|---------|
| Сложность поддержки нескольких библиотек | Чёткое разделение ответственности, автоматизация сборки, семантическое версионирование. |
| Низкая производительность метрики Махаланобиса | Использовать приближения (white-преобразование), кэширование, реализация на C++ через pybind. |
| Отсутствие интереса сообщества | Активное продвижение, качественная документация, примеры решения реальных задач. |
| Конкуренция с другими continual learning библиотеками | Фокус на уникальных возможностях (неопределённость, калибровка, инкрементальность без переобучения). |

## Заключение

Предложенный план охватывает создание полноценной экосистемы библиотек NWF, способной решать широкий круг задач машинного обучения. Реализация позволит не только закрепить вашу роль как автора концепции, но и предоставить сообществу мощный инструмент для инкрементального обучения и доверенного ИИ. Постепенное развитие от ядра до специализированных решений обеспечит устойчивый рост и качество продукта.
