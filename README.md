### Основные преимущества NWF по итогам исследования

1. **Нет катастрофического забывания** — при добавлении новых классов точность на старых остаётся высокой (86% после трёх задач против 18% у EWC и 54% у iCaRL).
2. **Встроенная уверенность** — ковариация позволяет калибровать вероятности (ECE = 0.03), что делает NWF пригодным для доверенного ИИ.
3. **Сверхкомпактность** — сжатие зарядов до 16 байт (49×) с потерей точности <1% благодаря PQ.
4. **Масштабируемость** — HNSW ускоряет поиск в 100 раз, позволяя работать с миллионами объектов.
5. **Семантическая интерпретируемость** — визуализация поля и интерполяция между объектами раскрывают внутреннюю структуру знаний.

NWF объединяет инкрементальность, доверие, компактность и скорость — уникальное сочетание для современных систем ИИ.

Научно-корректная реализация **Нейровесовых Полей (NWF)** — нового подхода к представлению семантики в латентном пространстве с встроенной неопределённостью.

## Что такое нейровесовые поля?

**Нейровесовые поля (NWF)** — это способ представления знаний в виде «зарядов» в латентном пространстве нейросети. Каждый заряд — это не просто точка (вектор), а точка **плюс мера неопределённости** вокруг неё. Это позволяет:

- **Инкрементально добавлять новые классы** без переобучения и без катастрофического забывания
- **Оценивать уверенность** в предсказании (калибровка)
- **Детектировать out-of-distribution** — данные, не похожие на обученные
- **Обновлять представление онлайн** при поступлении новых наблюдений (фильтр Калмана)

### Аналогия

Представьте, что каждый объект (например, изображение цифры) — это «заряд» в пространстве смыслов. У заряда есть центр (где он «находится») и «размытие» (насколько мы уверены в его положении). Близкие заряды притягиваются (схожие объекты), далёкие — отталкиваются. Поиск похожих объектов — это поиск ближайших зарядов по специальной метрике (Махаланобиса), учитывающей не только положение, но и неопределённость.

### Отличие от FAISS и эмбеддингов

| Подход | Что хранит | Добавление класса | Неопределённость |
|--------|------------|-------------------|------------------|
| **FAISS / эмбеддинги** | Точка z | Переобучение | Нет |
| **NWF** | Заряд (z, Sigma) | Добавить в индекс | Да |

## Основные концепции

### 1. Заряд (Charge)

**Заряд** — пара `(z, Sigma)`:
- `z` — центр в латентном пространстве (D-мерный вектор)
- `Sigma` — ковариационная матрица неопределённости (в реализации — диагональная)

VAE даёт нам `(mu, log_var)`; мы полагаем `z = mu`, `Sigma = diag(exp(log_var))`.

### 2. Расстояние Махаланобиса

Обычное евклидово расстояние не учитывает, что в одних направлениях вариативность больше, в других — меньше. **Махаланобис** масштабирует различие по «размаху» каждого измерения:

```
d² = (z_q - z_i)ᵀ Σ⁻¹ (z_q - z_i)
```

Для диагональной Sigma это сводится к `Σ (z_q - z_i)² / σ²` по каждой координате.

### 3. Симметричная метрика

При поиске учитываются неопределённости и запроса, и индекса. Используется `Σ_comb = Σ_q + Σ_i`:

```
d² = (z_q - z_i)ᵀ (Σ_q + Σ_i)⁻¹ (z_q - z_i)
```

Это устойчивее к шуму и лучше соответствует вероятностной интерпретации.

### 4. Потенциал Φ(z)

Сумма вкладов всех зарядов в точке запроса:

```
Φ(z_q) = Σᵢ exp(-d²(z_q, z_i) / 2)
```

Чем выше потенциал — тем «привычнее» запрос для модели (меньше вероятность OOD).

### 5. Kalman-кодировщик

Вместо одного прохода через VAE можно итеративно уточнять `(z, Sigma)` по наблюдению, минимизируя свободную энергию. Это даёт:
- Более точную реконструкцию
- Онлайн-обновление при новых кадрах (дрейф концепций)

## Структура проекта

```
nwf-research/
├── nwf/                    # Ядро библиотеки
│   ├── core.py             # Заряд, Махаланобис, потенциал
│   ├── vae_encoder.py      # VAE для получения (z, Sigma)
│   ├── storage.py          # Хранилище зарядов, k-NN поиск
│   ├── kalman.py           # Kalman-кодировщик (EKF)
│   ├── confidence.py       # Метрики уверенности
│   ├── inference.py        # MAP-оценка из VAE
│   ├── hnsw_index.py       # HNSW для ускорения поиска
│   ├── pq_index.py         # Product Quantization
│   ├── baselines.py        # FAISS (L2), HDC
│   └── continual.py        # EWC, iCaRL
├── experiments/            # Эксперименты 00-10
├── results/                # Результаты и графики
├── OTCHET_EKSPERIMENTOV.md
└── OTCHET_DETALNYY_SRAVNENIE.md   # Детальное сравнение с baseline
```

## Быстрый старт

```powershell
cd nwf-research
$env:PYTHONPATH = (Get-Location).Path

# 1. Обучить VAE
python train_vae.py --epochs 15

# 2. Запустить эксперимент сжатия (NWF vs FAISS)
python experiments/01_compression/run.py --metric symmetric --plot

# 3. Инкрементальное обучение (NWF vs EWC vs iCaRL)
python experiments/03_incremental/run.py --plot
```

Опции: `--metric symmetric`, `--use_kalman`, `--use_pq`

Интеграционная проверка: `python run_all_checks.py`
